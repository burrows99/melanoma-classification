The Clinical Challenge

Why Melanoma?

  • Causes 75%+ of skin cancer deaths despite being < 5% of cases
  • 5-year survival: 98%+ at Stage I vs 25% for metastatic
  • Early detection is critical

The Core Problem

  • Class imbalance: 13.6% malignant
  • All-benign predictor = 86.4% accuracy
  • But catches zero cancers
  • Accuracy is misleading on imbalanced data

Our Approach

  • 🔬 Dual-branch architecture: EfficientNet-B0 (image) + MLP (patient metadata)
  • ⚖️ Focal loss: Down-weight easy negatives, focus on hard malignant cases
  • 📊 Primary metric: F1 score (balances catching cancers vs false alarms)
  • 🛡️ OOD detection: Mahalanobis distance for out-of-distribution inputs

Dataset & Preprocessing

SIIM-ISIC 2020 Dataset

  • 37,648 dermoscopic images (256×256)
  • 13.6% malignant prevalence
  • 80/20 stratified split (seed=42)
  • Identical split across all experiments

Metadata Features (14-d)

  • age_approx: continuous (z-score normalized)
  • sex: binary (one-hot)
  • anatom_site: 6 categories (one-hot)
  • Median/mode imputation for missing values

Image Augmentation Pipeline

Training:
  • CLAHE (contrast enhancement)
  • Random blur
  • Horizontal/vertical flip (p=0.5)
  • Rotation ±15°
  • Color jitter
  • Random erasing
  • ImageNet normalization
Validation:
  • Resize to 256×256
  • ImageNet normalization
  • No augmentations (clean evaluation)

Model Architecture: Dual-Branch Late Fusion

EfficientNet-B0 (1280-d) + Metadata MLP (64-d) → Concatenation → Linear Head (1-d logit)

Why EfficientNet-B0?

  • 5.3M parameters (lightweight)
  • 77.1% ImageNet accuracy
  • Depthwise separable convs (8× cheaper)
  • SE attention + compound scaling
  • ISIC 2020 winner used EfficientNet ensemble

Why Late Fusion?

  • Image & metadata have different "languages"
  • Process separately → combine at decision
  • Not concatenate to pixels (age has no spatial meaning)
  • Metadata MLP: 14 → 128 → 64 (only 10k params!)
Fusion: [Image Features 1280-d] ⊕ [Metadata Features 64-d] = 1344-d → Linear → Sigmoid

Handling Class Imbalance: Focal Loss

The Problem

  • 6.4:1 ratio of benign to malignant samples
  • Standard cross-entropy: model can ignore minority class and still minimize loss
  • Easy negatives dominate gradients

Focal Loss Solution

FL(pt) = -αt · (1 - pt)γ · log(pt)

α (alpha) = 0.864

Derived from class proportion:

α = 1 - 0.136 = 0.864

Upweights malignant examples

γ (gamma) = 2.0 (default)

Modulates loss for easy vs hard examples:

  • High γ → Strong suppression of easy negatives
  • Low γ → Less suppression, more balanced
  • Experiment 3 tests γ=1.5

Key Insight

(1 - pt)γ — When model is confident and correct (p→1), loss → 0.
When model is wrong (p→0), loss is amplified. Forces focus on hard cases.

Checkpoint Metric
F1 = Harmonic Mean of Precision & Recall
F1 = 2 × (Precision × Recall) / (Precision + Recall)
In this project: F1 balances catching melanomas (recall) against avoiding false alarms (precision). A model that predicts everything as malignant has 100% recall but terrible precision—F1 penalizes this imbalance.

Training Loop

Per Epoch:
  1. Forward pass through dual-branch model
  2. Compute focal loss
  3. Backpropagate gradients
  4. Optimizer step (Adam or AdamW)
  5. Scheduler step (if enabled)
  6. Validate on held-out set
  7. Save checkpoint if val F1 improves
Best Checkpoint Selection:
  • Optimize for validation F1
  • Not accuracy (86.4% baseline is misleading)
  • Not recall alone (100% recall = predict all malignant)
  • F1 balances precision and recall

LR(t) = LRmin + ½(LRmax - LRmin)(1 + cos(πt/T))

Smoothly decays learning rate — lets model settle into sharper minima.

Key Metrics Explained

F1 Score — Primary Optimization Metric
F1 = 2 × (Precision × Recall) / (Precision + Recall)

In this project: F1 measures the balance between catching melanomas and avoiding false alarms.
Higher F1 = better trade-off between not missing cancers and not overwhelming dermatologists with benign referrals.
Recall (Sensitivity) — Clinical Priority
Recall = TP / (TP + FN) = % of true melanomas correctly identified

In this project: Recall = % of melanomas we catch.
A recall of 93.14% means we identify 93.14% of actual melanomas.
High recall is critical: Missing a melanoma (FN) sends a patient home with undetected cancer.
FN (False Negatives) — Missed Melanomas
FN = Patients with melanoma incorrectly predicted as benign

In this project: FN = number of missed cancers in validation set (~1,024 malignant samples).
Baseline FN=80 means 80 patients sent home with undetected melanoma.
Lower FN = fewer missed cancers = better clinical outcome.
FP (False Positives) — Unnecessary Biopsies
FP = Patients with benign lesions incorrectly predicted as malignant

In this project: FP = number of unnecessary referrals/biopsies in validation set (~6,506 benign samples).
Baseline FP=145 means 145 benign patients flagged for further investigation.
Lower FP = fewer unnecessary procedures, but not at the cost of missing cancers.
The Trade-off: FN reduction (catching more cancers) often increases FP (more false alarms).
F1 optimizes the balance. A model with 100% recall but 10,000 FP is useless clinically.

Out-of-Distribution Detection

OOD detection intercepts non-dermoscopic inputs before trusting predictions.

Mahalanobis Distance

d²(x) = (x - μ)T Σ-1 (x - μ)
How It Works:
  • Compute μ and Σ from validation embeddings
  • Threshold: μ + 3σ (empirical)
  • Ranges: 2637 (Exp 3) to 3248 (Baseline)
  • Flag inputs far from training distribution
Why It Matters:
  • Non-dermoscopic images can trigger false positives
  • Example: Koala photo → 85.8% malignant
  • Distance: 5124 >> threshold 2652
  • OOD warning prevents misleading confidence
Clinical Safety:
Without OOD detection, the model would confidently predict "malignant" on any image. The Mahalanobis distance acts as a safety gate — if the input is too different from training data, the system warns the user instead of giving a potentially wrong prediction.

Error Analysis: False Negatives & False Positives

False Negatives (~7-8.5% of malignant)

What the model misses:
  • Amelanotic melanomas — lack pigment
  • Small lesions — <6mm diameter
  • Early superficial spreading — subtle features
  • Lack typical ABCDE signs
  • CNN relies on pigmented features
FN = Missed Melanomas
Each FN is a patient sent home with undetected cancer. Exp 1 reduces FN from 80 to 70 — 10 fewer missed cancers.

False Positives (~1.4-2.8% of benign)

What triggers false alarms:
  • Dysplastic naevi — atypical but benign
  • Atypical Spitz naevi — irregular features
  • Seborrhoeic keratoses — irregular borders
  • Visually similar to melanoma
  • Lower cost than missed melanoma
FP = Unnecessary Biopsies
Each FP is an unnecessary referral. Higher FP is acceptable if it means catching more cancers. The F1 score balances this trade-off.

Why Not ViT / ResNet-50?

Vision Transformer (ViT):
  • No translation equivariance
  • Needs 10-100× more data
  • Not suited for ~37k images
ResNet-50:
  • 25.6M params (4.8× more)
  • 76.1% ImageNet (worse than B0)
  • No compound scaling benefit
